#include <stdio.h>
#include <limits.h> // INT_MAX, UINT_MAX 등이 정의됨

int main() {
    // 1. Signed Integer 실험
    int s_max = INT_MAX;
    printf("[Signed Int 실험]\n");
    printf("최댓값 (INT_MAX) : %d\n", s_max);
    printf("최댓값 + 1       : %d\n", s_max + 1);

    // 2. Unsigned Integer 실험
    unsigned int u_max = UINT_MAX;
    printf("\n[Unsigned Int 실험]\n");
    printf("최댓값 (UINT_MAX): %u\n", u_max);
    printf("최댓값 + 1       : %u\n", u_max + 1);

    return 0;
}

원인
비트의 한계: int는 32비트 공간만 가집니다. INT_MAX는 2진수로 0111...1111입니다. 여기에 1을 더하면 1000...0000이 됩니다.

해석의 차이 (Signed): signed 환경에서 맨 앞 비트(MSB)가 1이 되면 컴퓨터는 이를 음수로 인식합니다. 그래서 가장 큰 양수에서 1을 더하면 갑자기 가장 작은 음수로 "점프"하게 됩니다. (이를 Wrapping이라고 합니다.)

자리올림수 버림 (Unsigned): 모든 비트가 1인 상태에서 1을 더하면 받아올림이 발생하여 33번째 비트가 생겨야 하지만, 32비트 공간을 넘어가므로 앞자리는 버려지고 남은 자리는 모두 0이 됩니다.
